# Requirements Archive: v1.0 Atelier Code Intelligence

**Archived:** 2026-05-23
**Status:** ✅ All 18 v1 requirements shipped

## v1 Requirements (All Complete)

### Foundation

- [x] **FNDN-01**: Agent receives cached, budget-packed responses on existing `code` operations with `cache_hit`, `tokens_saved`, and `provenance` metadata. — *Phase 1, Validated*
- [x] **FNDN-02**: Agent can query symbol intelligence through routed precomputed backends behind the existing `code` surface, starting with SCIP when an index is available. — *Phase 1, Validated*

### Navigation

- [x] **NAVG-01**: Agent can search symbols with hardened defaults for snippets, ranking, and provenance on the existing `code op="search"` path. — *Phase 1, Validated*
- [x] **NAVG-02**: Agent can find usages of a symbol through `code op="usages"` without falling back to ad hoc text search by default. — *Phase 2, Validated*
- [x] **NAVG-03**: Agent can inspect callers and callees for a symbol through `code op="callers"` and `code op="callees"`. — *Phase 3, Validated*
- [x] **NAVG-04**: Agent can search and resolve code intelligence across supported multi-repo workspaces with repo-aware results. — *Phase 6, UAT Passed 2026-05-23*

### Editing & Discovery

- [x] **DISC-01**: Agent can apply symbol-scoped edits through the existing `edit` tool using a rich `kind="symbol"` descriptor. — *Phase 2, Validated*
- [x] **DISC-02**: Agent can run structural code search through `code op="pattern"` with tree-sitter-aware matching instead of regex-only search. — *Phase 2, Validated*
- [x] **DISC-03**: Agent can fall back to semantic symbol search over function-level embeddings when name-first retrieval is insufficient. — *Phase 3, Validated*
- [x] **DISC-04**: Agent can recall symbol-linked memory through the existing memory/code surfaces to recover prior context with low token overhead. — *Phase 3, Validated*
- [x] **DISC-05**: Agent can distinguish external dependency symbols from workspace symbols in code search results. — *Phase 6, UAT Passed 2026-05-23*

### History & Scale

- [x] **HIST-01**: Agent can search deleted or renamed symbols and filter historical results by time window or author. — *Phase 4, Validated*
- [x] **HIST-02**: Agent can inspect blame and churn metadata for a symbol to judge ownership and stability before editing. — *Phase 4, Validated*
- [x] **SCAL-01**: Agent can route large-repo search workloads through a validated scale backend once the build-vs-integrate checkpoint clears it. — *Phase 5, Validated*
- [x] **SCAL-02**: Agent can surface supported cross-language reference edges with confidence scoring for the planned Python/C, subprocess, and dynamic-import cases. — *Phase 5, Validated*

### Enablement

- [x] **ENBL-01**: Agent gets first-context bootstrap and prefetch behavior that warms the most relevant code-intel state before the first retrieval-heavy task. — *Phase 6, UAT Passed 2026-05-23*
- [x] **ENBL-02**: Maintainers have code-intel documentation, validation guidance, and scorecard metrics that explain when to use `code`, `read`, `search`, and symbol edits. — *Phase 7, Complete 2026-05-23*
- [x] **ENBL-03**: Maintainers have a documented build-vs-integrate decision record before large-repo backend work proceeds. — *Phase 5, Validated*
