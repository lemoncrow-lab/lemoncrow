<objective>
Land the read-context minifier and the reader-vs-writer dispatch in `PhaseRunner`:
the reader profile passes file bodies through `minify_source(text, lang)` before
they enter the conversation; the writer profile reads exact bytes; each minified
read records a `MinificationDelta` into `PhaseCacheStats.minify_deltas` so the
benchmark (13-04) can attribute savings separately from cache reuse. Satisfies
LINEAR-03 per D-09, D-10, D-11.

Purpose: Without language-safe minification and per-read telemetry, the
benchmark cannot separate cache-reuse savings (continue_from) from minification
savings (D-17), and writer reads risk silently corrupting source.

Output: `minify.py` utility, `MinificationDelta` model, reader/writer dispatch
in PhaseRunner, and two green pytest modules covering the five LINEAR-03 cases.
</objective>

|----------|-------------|
| read-tool body → reader-profile message list | minifier mutates string before model ingestion; any semantic alteration is a correctness bug |
| read-tool body → writer-profile message list | must transit byte-identically; minifier MUST NOT be invoked |
| minify_source pure function | accepts arbitrary text including hostile inputs; must not eval or import |

| T-13-02 | Tampering | `minify_source` transform | mitigate | string-only regex transforms; no `exec`/`eval`/`compile`; whitespace-significant set forces the safest path for Python/YAML/Makefile; `test_python_semantics_preserved` parses with `ast` to prove equality; `test_yaml_semantics_preserved` loads with `yaml.safe_load` to prove structural equality |
