  <action>
    Create `benchmarks/ab/runner.py`. This is the `__main__` entry-point for
    `python -m ab.runner` (and equivalently `python -m benchmarks.ab.runner` from repo root).

    MODULE DOCSTRING: Include the CLI usage example from AB-01.

    IMPORTS (stdlib only at module level; terminalbench imports inside main() to keep
    startup fast and avoid circular import at package init time):
    ```
    import argparse, datetime, json, os, sys
    from pathlib import Path
    ```

    ── SECTION 1: load_suite_tasks ──────────────────────────────────────────────
    Implement `load_suite_tasks(suite: str, n_tasks: int) -> list[str]`:
    - Only supported suite is `"terminalbench"`.
    - Locate tasks.yaml relative to this file:
      `Path(__file__).parent.parent / "terminalbench" / "tasks.yaml"`
    - Import `yaml` and load the file. Extract `data["tasks"]` (a list of task ID strings).
    - Return `tasks[:n_tasks]`.
    - Raise `ValueError(f"Unknown suite: {suite!r}")` for any other suite name.

    ── SECTION 2: write_config ───────────────────────────────────────────────────
    Implement `write_config(out_dir: Path, config: dict) -> None`:
    - Write `out_dir / "config.json"` atomically (tmp → os.replace pattern).
    - The config dict captures all parsed CLI arguments as a plain JSON-serialisable dict.

    ── SECTION 3: run_cell ──────────────────────────────────────────────────────
    Implement `run_cell(task_id, mode, rep, raw_dir, trial_dir, model, dataset_name, dataset_version) -> bool`:
    - Imports inside function: `from terminalbench.agent_adapter import run_terminalbench_trial`
    - Compute `dest = raw_dir / f"{task_id}__{mode}__rep{rep}.json"`.
    - SKIP: If `dest.exists()`, print `f"  skip {dest.name} (already done)"` and return True.
    - Run: call `run_terminalbench_trial(task_id, bench_mode=mode, rep=rep,
        out_dir=trial_dir / f"{task_id}__{mode}__rep{rep}",
        model=model, dataset_name=dataset_name, dataset_version=dataset_version)`.
    - ATOMIC WRITE: Serialize `result.to_dict()` with `json.dumps(..., indent=2, default=str)`.
      Write to `tmp = dest.with_suffix(".tmp")` then `os.replace(tmp, dest)`.
    - Return `not result.is_error`.

    ── SECTION 4: main ──────────────────────────────────────────────────────────
    Implement `main() -> None` with `argparse.ArgumentParser(prog="ab.runner")`:

    Arguments (all matching AB-01 exactly):
    - `--suite`  str, required, help="Benchmark suite name (e.g. terminalbench)"
    - `--tasks`  int, default=10, help="Number of tasks to select from suite"
    - `--n`      int, default=5, help="Number of repetitions per cell"
    - `--models` str, required, help="Claude model slug (e.g. claude-sonnet-4-5)"
    - `--modes`  str, default="on,off", help="Comma-separated list of bench modes"
    - `--out`    str, required, help="Output directory (e.g. bench/runs/<run-id>/)"
    - `--seed`   int, default=42, help="Random seed for task ordering (AB-04)"

    BODY of main():
    1. Parse args. Split `args.modes` on comma → `modes: list[str]`.
    2. `out_dir = Path(args.out)` and derive `run_id = out_dir.name`.
    3. `raw_dir = out_dir / "raw"` and `trial_dir = out_dir / "trials"`.
       Create both with `mkdir(parents=True, exist_ok=True)`.
    4. Load tasks: `task_ids = load_suite_tasks(args.suite, args.tasks)`.
    5. Load dataset meta from tasks.yaml to get `dataset_name` and `dataset_version`
       (re-read the yaml or accept the defaults "terminal-bench-core" / "0.1.1").
    6. Write config snapshot:
       ```python
       config = {
           "run_id": run_id,
           "suite": args.suite,
           "tasks": task_ids,
           "n_reps": args.n,
           "model": args.models,
           "modes": modes,
           "seed": args.seed,
           "started_at": datetime.datetime.utcnow().isoformat() + "Z",
       }
       write_config(out_dir, config)
       ```
    7. Build schedule: `from ab.schedule import build_schedule`
       `schedule = build_schedule(task_ids, modes, args.n, seed=args.seed)`
    8. Print header: total cells, how many already completed (count existing dest files).
    9. Iterate `for i, (task_id, mode, rep) in enumerate(schedule, 1)`:
       - Print progress: `f"[{i}/{len(schedule)}] {task_id} [{mode}] rep{rep}"`.
       - Call `run_cell(...)`. On exception, print the error and continue (do not abort sweep).
    10. Assemble summary:
        ```python
        from ab.aggregate import compute_summary
        summary = compute_summary(run_id, raw_dir)
        summary_path = out_dir / "summary.json"
        tmp = summary_path.with_suffix(".tmp")
        tmp.write_text(json.dumps(summary, indent=2))
        os.replace(tmp, summary_path)
        print(f"Summary written to {summary_path}")
        ```
    11. Print a brief pass-rate table per cell at the end (cell_key, passed/total, CI).

    ADD at module bottom:
    ```python
    if __name__ == "__main__":
        main()
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args → runner | User-supplied --out path; --tasks count; --seed |
| raw/ JSON files → aggregate | Runner writes these; must survive partial writes |
| terminalbench trial subprocess → runner | Docker subprocess; exceptions must not abort sweep |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-02-01 | Tampering | raw/ atomic write | mitigate | tmp → os.replace pattern; .tmp files are never treated as complete cells |
| T-03-02-02 | Denial of Service | trial exception propagation | mitigate | Wrap run_cell in try/except in main loop; print error, continue sweep |
| T-03-02-03 | Elevation of Privilege | --out path traversal | accept | Benchmark runner is developer-only tooling; not exposed to untrusted users |
| T-03-02-04 | Information Disclosure | config.json stores model slug | accept | config.json is internal benchmark metadata; no API keys stored |
| T-03-SC | Tampering | pip/uv installs | accept | No new packages; yaml is already in pyproject.toml (pyyaml>=6.0) |
</threat_model>
