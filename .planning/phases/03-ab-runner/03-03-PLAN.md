    Import: `from ab.schedule import build_schedule`

    Test `test_schedule_length`:
    - `schedule = build_schedule(["t1","t2","t3"], ["on","off"], 5, seed=42)`
    - `assert len(schedule) == 3 * 2 * 5  # 30`
    - `assert all(isinstance(t, tuple) and len(t) == 3 for t in schedule)`

    Test `test_schedule_determinism_ab04`:
    - Call `build_schedule(["a","b","c"], ["on","off"], 3, seed=42)` twice.
    - `assert s1 == s2  # same seed → same order`
    - Call with `seed=99`; `assert s_other != s1  # different seed → different order`
      (Note: theoretically could collide but won't for these short lists)

    Test `test_schedule_rep_by_rep_interleaving_ab02`:
    - `schedule = build_schedule(["t1","t2"], ["on","off"], 4, seed=42)`
    - Extract rep sequence: `reps = [r for _,_,r in schedule]`
    - Verify rep-by-rep invariant: for each rep N in 2..4, all indices of rep N must
      come after all indices of rep (N-1):
      ```python
      for n in range(2, 5):
          last_prev = max(i for i,(_,_,r) in enumerate(schedule) if r == n-1)
          first_curr = min(i for i,(_,_,r) in enumerate(schedule) if r == n)
          assert first_curr > last_prev
      ```

    Test `test_schedule_contains_all_cells`:
    - `tasks = ["x","y"]`, `modes = ["on","off"]`, `n_reps = 3`
    - Build schedule; verify every (task, mode, rep) combination appears exactly once:
      ```python
      from itertools import product
      expected = {(t, m, r) for t, m, r in product(tasks, modes, range(1, n_reps+1))}
      assert set(schedule) == expected
      ```

    Test `test_schedule_modes_not_shuffled`:
    - `schedule = build_schedule(["t1"], ["on","off"], 2, seed=42)`
    - For each rep, the two entries should appear in modes order ("on" before "off"):
      ```python
      for rep in [1, 2]:
          rep_entries = [(t,m,r) for t,m,r in schedule if r == rep]
          modes_in_rep = [m for _,m,_ in rep_entries]
          assert modes_in_rep == ["on","off"]
      ```

    Test `test_wilson_not_normal_approximation`:
    - Normal approximation for k=0,n=5 would give lo=0.0, hi=0.0. Wilson gives hi>0.
    - `_, hi = wilson_score_ci(0, 5)`
    - `assert hi > 0.0, "Wilson CI must give non-zero upper bound for k=0,n=5 (unlike normal approx)"`
