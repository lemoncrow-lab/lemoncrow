    - Length == len(tasks) * len(modes) * n_reps  (2 * 2 * 3 = 12 in example)
    - Outer loop is rep: all rep-1 entries appear before any rep-2 entry (AB-02 interleaving)
    - Within each rep, task order is a seeded shuffle — same seed → same shuffle each rep
    - build_schedule(..., seed=42) called twice returns identical list (AB-04 determinism)
    - build_schedule(..., seed=42) vs seed=99 likely returns different order (different shuffle)
    - modes order within each (rep, task) group is preserved as passed (not shuffled)
  </behavior>

    1. Create `benchmarks/ab/__init__.py` with module docstring and `__version__ = "0.1.0"`.

    2. Create `benchmarks/ab/schedule.py`:

       Module docstring: "Interleaved A/B execution schedule — AB-02 + AB-04."

       Implement `build_schedule(task_ids, modes, n_reps, seed=42) -> list[tuple[str, str, int]]`:
       - Accept task_ids: list[str], modes: list[str], n_reps: int, seed: int = 42.
       - Return type: list[tuple[str, str, int]] where each tuple is (task_id, mode, rep).
       - Use `random.Random(seed)` (not the global `random` module — avoids global state contamination).
       - For each rep in range(1, n_reps + 1):
         - Create a shuffled copy of task_ids using `rng.sample(task_ids, len(task_ids))`.
         - For each task_id in the shuffled list, for each mode in modes (in passed order):

       - IMPORTANT: Re-use the same `rng` object across all reps so each rep's shuffle is
         deterministic from the seed but distinct from other reps. Do NOT re-seed per rep.

<task type="auto" tdd="true">
  <name>Task 2: aggregate.py — Wilson CI + summary assembly (AB-05)</name>
  <files>benchmarks/ab/aggregate.py</files>
  <behavior>
    - wilson_score_ci(0, 0) == (0.0, 1.0)  (degenerate case — no division by zero)
    - wilson_score_ci(5, 5) returns (lower, 1.0) where lower > 0.5  (high-success CI)
    - wilson_score_ci(0, 5) returns (0.0, upper) where upper < 0.5  (all-fail CI)
    - wilson_score_ci(3, 5) returns tuple where 0 ≤ lower ≤ upper ≤ 1
    - wilson_score_ci(3, 5) does NOT store p_hat = 0.6 — only raw (k, n) stored (AB-05)
    - compute_summary reads raw/ dir, counts passed (grader_is_resolved == True), returns dict
    - compute_summary output has keys: run_id, generated_at, cells
    - Each cell key is "{task_id}__{mode}" and value has passed, total, ci_lower, ci_upper
  </behavior>
  <action>
    Create `benchmarks/ab/aggregate.py`:

    Module docstring: "Wilson-score CI and per-cell summary assembly — AB-05."

    Imports: `import json, math, datetime` from stdlib; `from pathlib import Path`.

    1. Implement `wilson_score_ci(k: int, n: int, z: float = 1.96) -> tuple[float, float]`:
       Copy the exact formula from the requirements verbatim:
       ```
       if n == 0: return (0.0, 1.0)
       p_hat = k / n
       denominator = 1 + z**2 / n
       center = (p_hat + z**2 / (2 * n)) / denominator
       margin = (z * math.sqrt(p_hat * (1 - p_hat) / n + z**2 / (4 * n**2))) / denominator
       return (max(0.0, center - margin), min(1.0, center + margin))
       ```

    2. Implement `_is_passed(record: dict) -> bool`:
       Returns True iff `record.get("grader_is_resolved") is True`.
       This is the single source of truth for pass/fail — do NOT use grader_verdict string.

    3. Implement `compute_summary(run_id: str, raw_dir: Path) -> dict`:
       - Scan raw_dir for all files matching `*.json` (not `*.tmp`).
       - Each filename follows `{task_id}__{mode}__rep{N}.json` — parse cell key as
         `"{task_id}__{mode}"` (everything before the final `__rep` segment).
         Use `stem.rsplit("__rep", 1)[0]` to extract the cell key robustly.
       - For each file, load JSON and call `_is_passed(record)`.
       - Accumulate `{cell_key: {"passed": int, "total": int}}` counters.
       - For each cell, call `wilson_score_ci(passed, total)` and store ci_lower, ci_upper
         rounded to 4 decimal places.
       - Return dict matching the summary.json schema:
         ```
         {
           "run_id": run_id,
           "generated_at": datetime.datetime.utcnow().isoformat() + "Z",
           "cells": {
             cell_key: {
               "passed": k, "total": n,
               "ci_lower": lower, "ci_upper": upper
             },
             ...
           }
         }
         ```
       - NEVER store `p_hat` in the output dict (AB-05). Only raw counts + CI bounds.

    Export `wilson_score_ci` and `compute_summary` in `__all__`.
  </action>
  <verify>

    raw.mkdir()

        (raw / f'hello-world__on__rep{rep}.json').write_text(json.dumps(rec))
    summary = compute_summary('smoke', raw)
    cell = summary['cells']['hello-world__on']
    assert cell['passed'] == 2
    assert cell['total'] == 3
    assert 'p_hat' not in cell, 'p_hat must NOT be stored (AB-05)'
    assert 'ci_lower' in cell and 'ci_upper' in cell
    print('aggregate.py OK')
"
