# Phase 18 Plan 01 — Shared tree-sitter tag extraction

## Goal

Add tree-sitter-backed tag extraction for the configured language set while preserving existing Python and legacy regex behavior.

## Tasks

1. Expose narrow public helper(s) from `treesitter_ast.py` for tree-sitter parser access, supported languages, and definition node kinds.
2. Add `_treesitter_tags(path, text, language)` in `tags.py`.
3. Implement line/byte helpers and grammar-aware name extraction for:
   - code languages: class/function/type/member declarations
   - bash: variables and function definitions
   - SQL: create table/view/index/function/alter table names
   - data languages: top-level JSON/YAML/TOML keys and tables
4. Route `extract_tags_from_text()`:
   - Python: stdlib `ast`
   - configured tree-sitter language: tree-sitter tags
   - legacy JS/TS/Go/Rust parser failure: regex fallback
   - other parser failure: `[]`
5. Add dedicated tag extraction tests.
