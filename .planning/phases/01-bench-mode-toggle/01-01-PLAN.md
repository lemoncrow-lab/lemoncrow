    with `from __future__ import annotations`):

    Module docstring: "Bench-mode singleton. ATELIER_BENCH_MODE=off produces the
    clean baseline arm — no Atelier routing, compaction, memory reads, or MCP
    tool substitution. Call bootstrap() once at process start (done by main()).
    is_off() lazy-bootstraps if needed so test code works without calling main()."

    Guard: `if _mode is not None: return` so double-calls are no-ops.

    is_off() -> bool: if _mode is None call bootstrap(); return _mode == BenchMode.OFF

    mode() -> BenchMode: if _mode is None call bootstrap(); return _mode  # type: ignore[return-value]

    make_arm_env(atelier_root: Path, *, mode: BenchMode | None = None) -> dict[str, str]:
    Start from `dict(os.environ)`. Set env["ATELIER_ROOT"] = str(atelier_root). Resolve
    mode_to_use: if mode arg is not None use it, elif _mode is not None use _mode, else
    use BenchMode.ON. Set env["ATELIER_BENCH_MODE"] = mode_to_use.value. Return env.

    __all__ = ["BenchMode", "bootstrap", "is_off", "make_arm_env", "mode"]

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| process env → bench singleton | ATELIER_BENCH_MODE is read from os.environ once at bootstrap(); value is trusted but must survive adversarial env (ATELIER_DEV_MODE=1 set by parent) |
| on-arm → off-arm filesystem | Shared ATELIER_ROOT would allow on-arm state to contaminate baseline reads |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | `mcp_tool_visible_to_llm` in environment.py | mitigate | bench.is_off() guard runs BEFORE is_dev_mode(); ATELIER_DEV_MODE=1 cannot bypass bench-off tool hiding |
| T-01-02 | Tampering | `_mode` singleton state across test cases | mitigate | Tests must monkeypatch `atelier.bench.mode._mode = None` in fixture teardown; documented in Plan 03 tests |
| T-01-03 | Information Disclosure | Shared ATELIER_ROOT between on-arm and off-arm subprocess runs | mitigate | make_arm_env(atelier_root) sets a subprocess-specific ATELIER_ROOT so each arm's ledger/memory writes go to an isolated tmp dir (MODE-06); enforced at runner level in Phase 2 |
| T-01-04 | Spoofing | ATELIER_BENCH_MODE value tampering mid-process | accept | bootstrap() is a no-op after first call; _mode is frozen at process start; no runtime override path exists by design |
| T-01-SC | Tampering | npm/pip/cargo installs | accept | This plan creates no new package-manager installs; bench/ uses stdlib only (os, enum, pathlib) |
</threat_model>
