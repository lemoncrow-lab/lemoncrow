# Plan 12-01: Cache-aware routing implementation

## Objective

Implement Phase 12 cache-aware routing by extending the existing model router with prefix-cache eviction economics, sticky follow-up routing, fail-open route-decision telemetry, and a deterministic local M2 benchmark.

## Goal-backward must_haves

<must_haves>
<truths>
D-01: Use the locked M2 design: compare `cache_eviction_cost_usd(prior_plan, current_plan, pricing)` against deterministic `estimated_quality_gain_usd`; preserve prior route when cache cost wins.
D-02: Implement cache-cost calculation as a pure `cache_cost.py` helper consuming `PrefixCachePlan` and `ModelPricing`, with safe stickiness bias when cache pricing is unknown.
D-03: Keep `ModelRouter.score()` backward-compatible; new router inputs are optional and do not break existing callers.
D-04: Add a default sticky window of three follow-up tool calls; sticky prior route wins when a prior route exists and remaining count is positive.
D-05: Keep sticky reset caller/runtime-owned at user-visible response boundaries; Phase 12 provides primitive and router behavior only.
D-06: Emit structured `route_decision` payloads with chosen/baseline tier, cache cost, estimated quality gain, decision, and remaining stickiness.
D-07: Keep telemetry fail-open so routing does not fail when event sinks fail.
D-08: Replace the M2 placeholder with a deterministic local replay benchmark proving at least 10% estimated cost reduction and zero quality-tier regressions.
</truths>
</must_haves>

- **CACHE-01 / D-03:** `ModelRouter.recommend()` exists with optional `prior_plan`, `current_plan`, `prior_route`, `stickiness_remaining`, and route-decision sink inputs; existing `score()` callers remain unchanged.
- **CACHE-02 / D-01 / D-02:** Router preserves the prior route when `cache_eviction_cost_usd(...) > quality_gain_usd_estimated` and switches when quality gain wins.
- **CACHE-03 / D-04 / D-05:** `stickiness.py` provides a default window of `3` and helpers for caller-owned sticky state/reset.
- **CACHE-04 / D-06 / D-07:** Cache-aware recommendations emit JSON-safe `route_decision` payloads through optional fail-open telemetry.
- **CACHE-05:** `cache_cost.py` and `stickiness.py` are public model-routing helpers exported from `model_routing/__init__.py`.
- **CQEVAL-03 / D-08:** `tests/benchmarks/context_quality/M2_routing.py` runs a deterministic slow replay benchmark proving `cost_reduction >= 0.10` and `quality_tier_regressions == 0`.

<threat_model>

## Threat Model

| Threat | Severity | Mitigation |
|---|---|---|
| Telemetry sink failure blocks routing | Medium | All route-decision sink/ledger errors must be caught and appended to recommendation reasons only when useful. Routing returns a recommendation regardless. |
| Unknown cache pricing creates invalid JSON or unsafe switching | Medium | `cache_eviction_cost_usd()` must use finite fallback values and never return `inf`/`nan`. |
| Cache-aware path regresses bench-off control arm | High | Preserve current `score()` bench-off behavior returning `None`; `recommend()` must delegate through `score()` and also return `None` in bench-off mode. |
| Benchmark optimizes cost while degrading quality tier | High | M2 benchmark must assert zero quality-tier regressions in addition to cost reduction. |

</threat_model>
