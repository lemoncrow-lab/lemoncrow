# Host Integration Matrix

This page summarizes the checked-in install surfaces for each supported host and
the safe operating contract each one gets from Atelier.

## Supported Hosts

| Host | Install path | Interface | Safe default | Enforcement contract | Trace coverage | Unsupported controls | Fallback |
| --- | --- | --- | --- | --- | --- | --- | --- |
| Claude Code | `docs/hosts/claude-code-install.md`, `integrations/claude/` | Generated AGENTS surface + MCP wrapper | Keep root entrypoints thin and use imported session traces for recovery | Prompt rules plus MCP tool-mode gating | High once Claude project exports are imported | No guaranteed host-side lifecycle hooks | Use `atelier` CLI and import sessions after execution |
| Codex CLI | `docs/hosts/codex-install.md`, `integrations/codex/` | Generated AGENTS surface + native Codex MCP registry + plugin | Prefer Atelier MCP tools for file I/O, search, edits, and shell work | Prompt rules plus Codex MCP registration/plugin install and eager MCP loading; native-tool fallback is policy-level, not hard host-side blocking | Medium-high after importing Codex sessions | No native hook bus, editor task bridge, or native-tool deny gate | Use native Codex tools only when Atelier is unavailable or returns `noop` |
| Cursor | `docs/hosts/cursor-install.md` | MCP server config + project-local rules | Prefer Atelier MCP tools over native Read/Grep for code insight | MCP visibility plus always-applied workspace Cursor rules | Medium; MCP-only path, no native hook bus | MCP cwd != workspace root, no global rules | Use `atelier` CLI and imported Cursor session traces |
| Copilot | `docs/hosts/copilot-install.md`, `integrations/copilot/` | VS Code MCP config + instructions + chat mode + tasks | Treat chat mode + MCP as primary, tasks as shell helpers | MCP visibility plus generated Copilot instructions that name native-tool fallbacks explicitly | Medium; chat traces are strong, shell task traces are weaker | `tasks.json` cannot invoke MCP directly | Use `atelier` CLI tasks and runtime evidence capture |
| opencode | `docs/hosts/opencode-install.md`, `integrations/opencode/` | Generated agent file + MCP wrapper + DB import | Make Atelier the default agent and lean on imported sessions when native controls are thin | Prompt rules plus default-agent selection and MCP tool-mode gating | Medium after local DB import | No stable hook/event API | Use direct Atelier CLI commands and session import |
| Hermes Agent | `docs/hosts/hermes-install.md` | YAML config (`$HERMES_HOME/config.yaml`) + toolset registration | Treat MCP as primary; keep `mcp-atelier` first in toolset CLI order | MCP registration plus toolset CLI ordering | Medium; MCP-only path with no hook bus | No CLI detection, YAML-only config | Use `atelier` CLI and manual trace recording |
| Antigravity | `docs/hosts/antigravity-install.md`, `integrations/antigravity/` | Workspace/user MCP config + generated AGENTS surface | Treat MCP as primary and use `agy` as the terminal companion | MCP registration plus generated Antigravity guidance and plugin metadata | Medium; live MCP is strong, host-native hooks are still thin | No first-class hook/event contract | Use explicit verification loops and trace recording |
