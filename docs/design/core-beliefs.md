# Core Beliefs

1. Repository knowledge beats prompt repetition.
2. Thin entrypoints scale better than giant instruction files.
3. Every durable rule should have a home in-repo and a path to enforcement.
4. Plans, decisions, and debt should be discoverable without external chat logs.
5. Agents ship faster when validation loops are explicit and reusable.
6. If a rule matters often, generate it or test it instead of asking for it again.
