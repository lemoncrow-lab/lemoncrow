## The big picture

Excalidraw separates **scene state** (the source of truth, plain element objects) from **rendering** (drawing those objects to HTML canvases). State changes mutate elements and bump a version counter; that invalidates caches and triggers a throttled re-draw onto layered canvases. RoughJS shapes are cached per-element so they're only regenerated when an element actually changes.

## Rendering: three layered canvases

Rather than one canvas, Excalidraw stacks three, each with a different update cadence:

- **`StaticCanvas`** (`packages/excalidraw/components/canvases/StaticCanvas.tsx:56`) — the committed scene: shapes, text, images. Drawn by `renderStaticScene` (`renderer/staticScene.ts:491` → `_renderStaticScene` at `:229`), which handles grid, frame clipping, and loops over visible elements calling `renderElement`.
- **`InteractiveCanvas`** (`components/canvases/InteractiveCanvas.tsx:24`) — transient overlays: selection handles, transform boxes, snap lines, binding highlights, cursors. Drawn by `renderInteractiveScene` (`renderer/interactiveScene.ts:2082`).
- **`NewElementCanvas`** (`components/canvases/NewElementCanvas.tsx:31`) — the element currently being drawn, before it's committed to the scene.

The per-element workhorse is **`renderElement`** (`packages/element/src/renderElement.ts:780`), which applies opacity/transform and dispatches by element type (rectangle, arrow, text, frame, image…).

**Scheduling** is a throttled `requestAnimationFrame`. `renderStaticSceneThrottled` (`staticScene.ts:482`) wraps the real render in `throttleRAF`, so multiple state changes within a frame collapse into one draw. The interactive layer runs its own RAF loop via an `AnimationController` (`renderer/animation.ts`). React's component re-render is what kicks these off — the canvases redraw when their props change.

## Shape generation & caching (the key perf trick)

Shapes are drawn sketchy via **RoughJS**, which is expensive to generate, so they're cached:

- **`ShapeCache`** (`packages/element/src/shape.ts:81`) is a `WeakMap<Element, ElementShape>`. `generateElementShape()` (`:118`) builds and caches a Drawable; `get()` (`:92`) reuses it if present and the theme matches; `delete()` (`:105`) evicts it.
- **`generateRoughOptions`** (`shape.ts:193`) maps element style props (stroke, fill, roughness) into RoughJS options.
- In `renderElement`, the cached shape is fetched and drawn with `rc.draw(shape)`.

Net effect: a shape is regenerated **only** when uncached, the element changed, or during export — not on every frame.

## Updating: scene state → mutation → invalidation → re-render

**The Scene** (`packages/element/src/Scene.ts`) is the source of truth. It holds all elements (`elementsMap`) plus derived non-deleted views (`nonDeletedElements` array and `nonDeletedElementsMap`, via `getNonDeletedElements()` / `getNonDeletedElementsMap()`). `triggerUpdate()` (`:303`) regenerates a `sceneNonce` and fires all listeners registered through `onUpdate()` (`:311`).

**Mutating an element** goes through `mutateElement()` (`packages/element/src/mutateElement.ts:37`), which edits the object in place and automatically bumps the versioning fields:
- `version` += 1 (`:139`)
- `versionNonce` = new random int (`:140`)
- `updated` = timestamp (`:141`)

If a change affects geometry (dimensions, points, fileId), it calls `ShapeCache.delete(element)` (`:136`) to evict the stale cached shape. `newElementWith()` (`:146`) is the immutable variant that returns a fresh copy with the same version bumping.

**The invalidation chain:** `Scene.mutateElement` → version changed → `triggerUpdate()` → `sceneNonce` changes → registered listeners fire.

**Connecting to React** (`packages/excalidraw/components/App.tsx`): the App registers `this.scene.onUpdate(this.triggerRender)` (`:3147`). `triggerRender` calls `setState({})` (`~:4655`) to force a React re-render. During `render()` (`:2104`), it pulls elements through the memoized **`Renderer`** (`packages/excalidraw/scene/Renderer.ts`). `_getRenderableElements()` (`:159`) is memoized on a `canvasNonce` built from `sceneNonce` (`:210`) — so when the nonce changes, the memo busts, renderable elements recompute, and the canvases redraw. Separately, `onChange` callbacks fire so consumers (collaboration, persistence) hear about updates.

## Versioning model (collaboration / undo)

Three fields on every element (`packages/element/src/types.ts:58`) drive reconciliation:
- **`version`** — sequential counter; higher wins when merging concurrent edits.
- **`versionNonce`** — random tiebreaker when two versions are equal (deterministic across clients).
- **`updated`** — last-modified timestamp.

New elements start at `version: 1`, `versionNonce: 0` (`newElement.ts:146`).
