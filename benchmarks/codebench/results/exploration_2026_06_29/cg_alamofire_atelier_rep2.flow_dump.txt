**Build**
- `Session` (default `AF`) is the entry point; wraps a `URLSession` + delegate + a serial `rootQueue`.
- `AF.request(url, method:parameters:encoding:headers:)` → creates a `RequestConvertible`/`RequestEncodableConvertible` that captures the params.
- On the `rootQueue`, `Session.perform` asks the convertible for a `URLRequest`:
  - `URLRequest` built from `URLComponents` (url, method, headers).
  - `ParameterEncoding` (`URLEncoding`/`JSONEncoding`) or `ParameterEncoder` (`URLEncodedFormParameterEncoder`/`JSONParameterEncoder`) mutates it — query string vs body, sets `Content-Type`.
  - `RequestAdapter` (via `RequestInterceptor`) gets a last async pass — inject auth headers, sign, etc.
- Result wrapped in a `DataRequest` (subclass of `Request`), state = `.initialized`.

**Send**
- `Session` calls `request.didCreateURLRequest`, then builds a `URLSessionTask` from the `URLRequest` and resumes it (respects `startRequestsImmediately`).
- The `SessionDelegate` receives `URLSession` callbacks on the delegate queue, routes them back to the `Request` on its `underlyingQueue`; bytes accumulate in the `DataRequest`'s buffer.
- `RequestInterceptor.retry` decides on failure: `.retry` / `.retryWithDelay` re-runs the whole adapt→task cycle, or `.doNotRetry`.
- Task completes → `Request` transitions to `.finished`.

**Validate**
- `.validate()` appends a closure to the request's `validators`; multiple allowed. Two built-in forms:
  - status code — default acceptable is `200..<300`; `.validate(statusCode:)` for a custom set.
  - `Content-Type` — matches response MIME against the request's `Accept` header (or an explicit list).
  - `.validate { request, response, data in ... }` for a custom `ValidationResult` (`.success`/`.failure(Error)`).
- Validators run before the response serializer. A failure sets the request's `error` to `AFError.responseValidationFailed(reason:)`.

**Serialize/deliver**
- `.responseDecodable(of:)` / `.responseData` / `.responseString` append a serializer that runs on a `serializationQueue` after validation.
- Serializer sees the validation error first (short-circuits), else decodes the buffered data.
- Final `AFDataResponse<T>` (request, response, data, metrics, `Result<T, AFError>`) dispatched to the caller's `queue` (default `.main`).

Typical chain:
```swift
AF.request("https://api.example.com/users",
           parameters: ["page": 1],
           encoding: URLEncoding.default)
  .validate(statusCode: 200..<300)
  .validate(contentType: ["application/json"])
  .responseDecodable(of: [User].self) { response in
      switch response.result {
      case .success(let users): …
      case .failure(let error): …   // AFError
      }
  }
```

Core pipeline: `convertible → URLRequest → ParameterEncoder → RequestAdapter → URLSessionTask → SessionDelegate → validators → serializer → AFDataResponse`. Interceptor sits at both adapt (outbound) and retry (failure) points; everything is marshaled through the `Session`'s serial `rootQueue` for thread-safety.
