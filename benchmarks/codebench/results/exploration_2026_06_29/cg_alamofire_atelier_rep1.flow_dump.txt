## Build
- Entry: `Session.request(_:method:parameters:encoding:headers:interceptor:)` — or pass a `URLRequestConvertible` directly (e.g. a `Router` enum).
- Convenience args get wrapped in a `RequestConvertible`/`RequestEncodableConvertible` that conforms to `URLRequestConvertible`. Its `asURLRequest()`:
  1. builds a base `URLRequest` from URL + method + headers,
  2. runs a **`ParameterEncoding`** (`URLEncoding`, `JSONEncoding`) or the newer **`ParameterEncoder`** (`URLEncodedFormParameterEncoder`, `JSONParameterEncoder`) to fold parameters into the request — query string vs. httpBody depending on encoder/method.
- Result is a `DataRequest` (or `DownloadRequest`/`UploadRequest`/`DataStreamRequest`) added to the `Session`.

## Send
- All state mutation runs on the Session's serial `rootQueue` for thread safety.
- `Request` gathers its initial `URLRequest`, then the **`RequestInterceptor`** pipeline runs `adapt(_:for:completion:)` — last chance to mutate (auth headers, etc.).
- Adapted request → `URLSessionTask` created on the Session's `URLSession`. The **`SessionDelegate`** receives all `URLSession` delegate callbacks and routes each to the owning `Request` via a `requestTaskMap`.
- Task `resume()`s (unless the request was created suspended). Response body accumulates on the delegate; metrics/errors captured.
- On task completion, **response serialization** (`.response`, `.responseDecodable(of:)`, `.responseData`, …) runs on a serialization queue, producing an `AFDataResponse<T>`.
- Failures can re-enter the interceptor's `retry(_:for:dueTo:completion:)` → `.retry` / `.retryWithDelay` / `.doNotRetry`, looping back to send.

## Validate
- `.validate()` inserted in the chain before serialization appends a `Validation` closure. Default checks:
  - **status code** in `200..<300`,
  - **Content-Type** compatible with the request's `Accept` header.
- Custom: `.validate { request, response, data in ... }` returning `.success`/`.failure(error)`, or `.validate(statusCode:)` / `.validate(contentType:)`.
- A failed check yields `AFError.responseValidationFailed(reason:)`, which propagates into the serialized response's `.result` (and can trigger retry).

Chain shape: `session.request(...).validate().responseDecodable(of: T.self) { response in ... }` — validate runs on the task result, serialization only proceeds on a passing validation.
