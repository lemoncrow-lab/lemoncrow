## plugin:atelier:atelier
Atelier replaces the grep→read→re-read loop.
- Lead with `code_search`: one call = ranked matches (top source inline, the rest as precise path:Lx-Ly pointers) + related_symbols + candidate_files. Inline source = already read. Never shell-grep workspace code or re-verify indexed results.
- Known path/symbols →  batch `read`: ONE call, files=[...], exact :Lx-Ly ranges, never the same file twice. Never cat/sed/head/tail.
- ALL edits in ONE `edit` edits[] array; prefer {path: 'f.py:Lx-Ly', new} over old/new.
- `bash` = execution only (tests, git, builds).
- Large output → a file, never inline prose.

[assistant] Django ORM query lifecycle — build lazily, compile on demand, execute once, cache:

**Build (no DB hit).** A `QuerySet` wraps a `Query` object (`django.db.models.sql.query.Query`). Each `filter`/`exclude`/`annotate`/`order_by` **clones** the QuerySet and mutates the clone's `Query`:
- `add_q(Q(...))` → resolves field names against the model, adds table aliases/JOINs (`Query.table_alias`, `join()`), builds the WHERE tree as a `WhereNode` of `Lookup` nodes (e.g. `Exact`, `In`).
- Field refs become `Col` expressions bound to an alias; `annotate` adds to `.annotations`, `values()` sets the select mask.
- All lazy — clone-and-accumulate, still zero queries.

**Trigger.** DB hit only on evaluation: iteration, `list()`, `len()`, `bool()`, slicing-with-step, `repr()`, pickling. These call `QuerySet._fetch_all()`.

**Compile.** `_fetch_all` runs `self._iterable_class` (`ModelIterable` / `ValuesIterable` / `FlatValuesListIterable` …), which calls `query.get_compiler(using=db)` → `SQLCompiler` (`django.db.models.sql.compiler`). `compiler.as_sql()`:
- `pre_sql_setup` → `get_select` (SELECT cols + annotations), `get_from_clause` (FROM + JOINs from the alias map), compiles the `WhereNode` to SQL, then GROUP BY / ORDER BY / LIMIT.
- Returns `(sql_string, params)` — a **parameterized** statement; values stay separate (SQL-injection safe). Dialect specifics (quoting, LIMIT syntax, casts) come from `connection.ops` per backend.

**Execute.** `compiler.execute_sql()` gets a cursor from the connection, `cursor.execute(sql, params)`, fetches rows.

**Map + cache.** The iterable class turns each row into a model instance (or tuple/dict). Results land in `QuerySet._result_cache` — re-iteration serves the cache, no re-query.

Key files: `db/models/query.py` (QuerySet), `db/models/sql/query.py` (Query), `db/models/sql/compiler.py` (SQLCompiler), `db/models/sql/where.py` (WhereNode), `db/backends/*/operations.py` (dialect).
