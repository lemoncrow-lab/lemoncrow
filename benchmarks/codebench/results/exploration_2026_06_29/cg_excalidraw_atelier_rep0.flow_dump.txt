## plugin:atelier:atelier
Atelier replaces the grep→read→re-read loop.
- Lead with `code_search`: one call = ranked matches (top source inline, the rest as precise path:Lx-Ly pointers) + related_symbols + candidate_files. Inline source = already read. Never shell-grep workspace code or re-verify indexed results.
- Known path/symbols →  batch `read`: ONE call, files=[...], exact :Lx-Ly ranges, never the same file twice. Never cat/sed/head/tail.
- ALL edits in ONE `edit` edits[] array; prefer {path: 'f.py:Lx-Ly', new} over old/new.
- `bash` = execution only (tests, git, builds).
- Large output → a file, never inline prose.

    visibleElements,
    selectedElements,
    frameToHighlight,

    if (!selectedElements.length) {
      return visibleElements;

    // frameToHighlight is defined

      deselectedElements,
      frameToHighlight.id,
    );

      return visibleElements;
    }

    return [
      ...deselectedElements.slice(0, insertionIndex),
      ...selectedElements,
      ...deselectedElements.slice(insertionIndex),
    ] as readonly T[];

That returns `{ elementsMap, visibleElements, canvasNonce }` and is passed as props to `<StaticCanvas>`. It never draws directly — props flow into the memoized canvas components.

**Renderer memoization** (`scene/Renderer.ts`):
- `_getRenderableElements` is `memoize`d on `canvasNonce` = `scene.getSceneNonce()` (+ newElement versionNonce). Same nonce → cached element/visible maps returned, no recompute.
- Builds a `RenderableElementsMap` (drops the new element + the text being edited), then `getVisibleCanvasElements` filters by `isElementInViewport` (viewport culling — only on-screen elements are painted).

**React re-render gate.** `StaticCanvas` uses `React.memo(areEqual)`: repaints only if `canvasNonce`, `scale`, `elementsMap`, `visibleElements`, or a whitelisted subset of `appState` (`getRelevantAppStateProps`) / `renderConfig` changed (`isShallowEqual`). On pass, a `useEffect` (runs every committed render) calls `renderStaticScene`.

**Scene paint** (`renderer/staticScene.ts`):
- `renderStaticScene` → `renderStaticSceneThrottled` = `throttleRAF(...)` — coalesced to one paint per animation frame.
- `_renderStaticScene` clears, then `visibleElements.forEach(renderElement)`, splitting passes: normal elements → iframe/embeddables on top → bound text with containers → frame clipping. Each element wrapped in `context.save()/restore()` and try/catch.

**Per-element draw — three-level cache** (`element/src/renderElement.ts`, `shape.ts`):
1. **`ShapeCache`** (`WeakMap<element, {shape, theme}>`) — the RoughJS `Drawable` geometry. `generateElementShape` regenerates only on cache miss / theme change / export.
2. **`elementWithCanvasCache`** (`WeakMap<element, canvas>`) — an offscreen per-element bitmap. `generateElementWithCanvas` regenerates when: no prior, zoom changed (unless `shouldCacheIgnoreZoom`), theme changed, bound-text version changed, image crop changed, containing-frame opacity changed, or arrow-with-label angle changed.
3. **`drawElementFromCanvas`** — blits the cached bitmap onto the scene ctx (rotate/translate), so steady-state repaint is just image copies, not re-tessellation.

`freedraw`/export path draws directly via `drawElementOnCanvas` instead of caching.

**Invalidation.** WeakMaps keyed by element identity, so a mutated element is a cache miss automatically. Explicit `ShapeCache.delete(element)` on edits (App.tsx ~L1513, 3090, 11921); image loads clear via `clearImageShapeCache`. Scene mutations bump `sceneNonce` → new `canvasNonce` → Renderer recompute + StaticCanvas repaint. Element `version`/`versionNonce` propagate through bound-text checks.

Net: React memo gate → nonce-memoized renderable set → viewport cull → RAF-throttled scene paint → shape cache → per-element bitmap cache → blit. Cheap updates (pan, selection) touch only the interactive layer; element edits invalidate that element's caches and repaint the static layer once per frame.
