        }
      } catch (error: any) {
        console.error(error);
      }
    });

  renderConfig.pendingFlowchartNodes?.forEach((element) => {
    try {
      renderElement(
        element,
        elementsMap,
        allElementsMap,
        rc,
        context,
        renderConfig,
        appState,
      );
    } catch (error) {
      console.error(error);
    }
  });

## Rendering: layered canvases

Three stacked `<canvas>` React components, each `React.memo` with a custom `areEqual`:
- **StaticCanvas** — committed elements (fills, strokes, text). `packages/excalidraw/components/canvases/StaticCanvas.tsx`
- **InteractiveCanvas** — selection boxes, handles, remote cursors, snap lines. Never memoized on cursor move.
- **NewElementCanvas** — the single element being actively drawn (draw-perf optimization).

Split exists so a selection change repaints only the cheap interactive layer, not every element.

## Update trigger chain

`mutateElement` (`packages/element/src/mutateElement.ts`) → `element.version++`, new `versionNonce`, **`ShapeCache.delete(element)`** (invalidates that element's cached shape) → `Scene.mutateElement` → `Scene.triggerUpdate()` sets `sceneNonce = randomInteger()` → `onUpdate` callback → App `setState` → `App.render()`.

`App.render()` → `Renderer.getRenderableElements({...})` (`scene/Renderer.ts`):
- memoized (`memoize`) on `canvasNonce = sceneNonce` — same nonce → cached result, no recompute.
- builds `elementsMap`, drops the currently-edited text element + `newElement`, then **viewport-culls** via `isElementInViewport` → `visibleElements`.

Result props flow to the canvas components. `areEqual` compares `canvasNonce / scale / elementsMap / visibleElements / appState` — unchanged → React skips, no repaint.

## Actual paint

StaticCanvas `useEffect` → `renderStaticScene` → **`renderStaticSceneThrottled` (`throttleRAF`, one paint per animation frame)** → `_renderStaticScene` (`renderer/staticScene.ts`):
1. bootstrap canvas (size, DPR scale, bg), `context.scale(zoom)`, grid,
2. per `visibleElement` → `renderElement(...)`; text-in-container and iframe-likes handled specially (embeddables drawn on top).

## Per-element cache (why it's fast) — `element/src/renderElement.ts`

Two WeakMaps keyed by element identity:
- **`ShapeCache`** (`element/src/shape.ts`) — rough.js `Drawable` geometry, checked against `theme`.
- **`elementWithCanvasCache`** — a pre-rendered **off-screen canvas per element**. `generateElementWithCanvas` regenerates only when `zoomValue`, `theme`, bound-text version, image crop, containing-frame opacity (or arrow-label angle) changed; otherwise `drawElementFromCanvas` just blits the cached bitmap.

So a scene mutation only busts the caches of the mutated elements (via `ShapeCache.delete`); everything else is copied from its cached canvas. Interactive layer redraws on its own nonces (`selectionNonce`, etc.) independent of element geometry.

Net: **identity + nonce/version invalidation → React memo gate → RAF throttle → per-element bitmap cache**, four layers that each avoid redundant work.
