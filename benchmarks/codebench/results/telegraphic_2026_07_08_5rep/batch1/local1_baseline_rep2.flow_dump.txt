[assistant] React components re-render when their parent re-renders by default — that's just how React works. When a parent re-renders, React recursively re-renders all its children, regardless of whether their props actually changed. This is usually fine and fast, but here's how to control it when it matters.

## Why it happens

React's default reconciliation re-renders the entire subtree below a component that updates. Props "not changing" doesn't stop it — React doesn't check, it just re-renders and then diffs the output against the DOM.

## How to stop unnecessary re-renders

**1. Wrap the child in `React.memo`**

This tells React to skip re-rendering when props are shallowly equal:

```jsx
const Child = React.memo(function Child({ name }) {
  return <div>{name}</div>;
});
```

Now `Child` only re-renders when `name` actually changes.

**2. Watch out for new object/function references**

`React.memo` does a *shallow* comparison. If you pass inline objects, arrays, or functions, they're recreated every render and break memoization:

```jsx
// ❌ new function + new object every parent render → memo is useless
<Child onClick={() => doThing()} style={{ color: 'red' }} />

// ✅ stable references
const onClick = useCallback(() => doThing(), []);
const style = useMemo(() => ({ color: 'red' }), []);
<Child onClick={onClick} style={style} />
```

**3. Move state down, or lift content up**

Often the real fix is structural. If only part of the parent needs the frequently-changing state, isolate it:

```jsx
// Instead of re-rendering the whole parent on every keystroke,
// put the input + its state in a small dedicated component.
```

Or use the **children-as-props** trick — children passed through don't re-render when the parent's own state changes:

```jsx
function Parent({ children }) {
  const [count, setCount] = useState(0);
  return <div onClick={() => setCount(c => c + 1)}>{children}</div>;
}
// <ExpensiveChild/> passed as children won't re-render on count change
```

## Before you optimize

Re-renders are often cheap — React re-rendering ≠ the DOM updating. Don't reach for `memo`/`useCallback` everywhere reflexively; it adds complexity and comparison cost. Profile first with the **React DevTools Profiler** ("Highlight updates" / record a render) to confirm a re-render is actually expensive before optimizing it.
