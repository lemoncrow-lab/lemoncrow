## How it minimizes final image size

The final image contains **only** three things: the Alpine Node runtime, production `node_modules`, and the compiled `dist/` — no source, no TypeScript compiler, no dev tooling, no build caches.

**The four stages:**
1. **`deps`** — `npm ci` installs *all* deps (needs devDependencies like `typescript` to build). Cached separately so it only re-runs when the lockfile changes.
2. **`build`** — reuses those deps and runs `npm run build` to compile TS → JS.
3. **`prod-deps`** — a *separate* `npm ci --omit=dev` producing a clean production-only `node_modules`. This is the key trick: rather than trying to prune the dev tree, we build a fresh one that never had devDependencies in it.
4. **`runtime`** — copies just the prod `node_modules` + `dist/` into a fresh `node:22-alpine` base. Everything else is discarded.

**Other size/quality choices:**
- `node:*-alpine` base (~50 MB vs ~1 GB for the default Debian image).
- `--mount=type=cache` for the npm cache — speeds up rebuilds *without* baking the cache into a layer.
- `.dockerignore` keeps the build context tiny and stops the host's `node_modules` from overwriting the container's.
- Runs as the non-root `node` user, sets `NODE_ENV=production`, and uses exec-form `CMD` so `SIGTERM` reaches Node for graceful shutdown.

## You'll likely need to adjust
- **`package.json` scripts** — assumes a `build` script (e.g. `"tsc"`) and that output lands in `dist/` with entry `dist/index.js`. Change the `COPY --from=build /app/dist` path and `CMD` if yours differ.
- **`EXPOSE 3000`** — set to your actual port.
- **Node version** — pin `node:22-alpine` to whatever your app targets.
