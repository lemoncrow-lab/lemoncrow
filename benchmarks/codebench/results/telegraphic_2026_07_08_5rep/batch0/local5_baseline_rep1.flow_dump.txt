```javascript
async function getUser(id) {
  const rows = await db.query('SELECT * FROM users WHERE id = ?', [id]);
  if (!rows.length) throw new Error('Not found');
  return rows[0];
}
```

This assumes `db.query` returns a promise that resolves to `rows`. A few notes depending on your driver:

**If `db.query` is still callback-based** (only takes a callback), promisify it first:

```javascript
const { promisify } = require('util');
const query = promisify(db.query).bind(db);

async function getUser(id) {
  const rows = await query('SELECT * FROM users WHERE id = ?', [id]);
  if (!rows.length) throw new Error('Not found');
  return rows[0];
}
```

**If you're on `mysql2/promise`**, `query` resolves to `[rows, fields]`, so destructure:

```javascript
async function getUser(id) {
  const [rows] = await db.query('SELECT * FROM users WHERE id = ?', [id]);
  if (!rows.length) throw new Error('Not found');
  return rows[0];
}
```

Key behavior changes:
- The `err` branch is gone — a rejected query now propagates automatically, so callers use `try/catch` (or `.catch()`) instead of checking an `err` argument.
- Errors are `throw`n rather than passed to a callback; the `Not found` case becomes a thrown error too. Callers who want to distinguish "not found" from a DB failure may prefer returning `null` instead of throwing.

Callers change from `getUser(1, (err, user) => …)` to `const user = await getUser(1)`.
