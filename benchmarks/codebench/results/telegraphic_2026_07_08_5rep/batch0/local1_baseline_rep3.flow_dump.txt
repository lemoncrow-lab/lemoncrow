[assistant] This is one of the most common React gotchas. The short answer: **passing an object literal as a prop creates a brand-new object on every render, so even a memoized child sees "different" props.**

## Why it happens

React compares props by **reference identity** (`Object.is`), not by deep value. When you write:

```jsx
<Child config={{ id: 1, name: "foo" }} />
// or
<Child style={{ margin: 10 }} data={someArray.map(...)} />
```

...that object/array is re-created fresh on every parent render. `{a:1} !== {a:1}` — different references — so the child re-renders even though the *contents* are identical.

The same applies to:
- **Inline objects/arrays**: `prop={{...}}` / `prop={[...]}`
- **Inline functions**: `onClick={() => ...}` (new function each render)
- **Derived arrays**: `.map()`, `.filter()`, `.sort()` in the render body

## Important caveat

By default, **`React.memo` is what makes this matter**. A plain child component *always* re-renders when its parent does, regardless of props. So there are two distinct questions:

1. If the child is **not** wrapped in `React.memo`, it re-renders because its parent did — the object prop is a red herring. Fix the parent, not the prop.
2. If the child **is** `React.memo`'d, the unstable object reference is defeating the memoization.

## Fixes

**Stabilize the object reference with `useMemo`:**
```jsx
const config = useMemo(() => ({ id, name }), [id, name]);
<Child config={config} />
```

**Stabilize callbacks with `useCallback`:**
```jsx
const handleClick = useCallback(() => doThing(id), [id]);
```

**Memoize the child** (only helps combined with stable props):
```jsx
const Child = React.memo(function Child({ config }) { ... });
```

**Or hoist truly-static objects out of the component** so they're created once:
```jsx
const STATIC_CONFIG = { id: 1, name: "foo" };
function Parent() { return <Child config={STATIC_CONFIG} />; }
```

## How to confirm the cause

- Add `<React.Profiler>` or use the **React DevTools Profiler** → "why did this render" highlights prop changes.
- Temporarily log or compare the prop reference across renders.
