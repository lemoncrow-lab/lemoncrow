[assistant] Common causes, roughly in order of likelihood:

## 1. It's actually normal — React re-renders children by default
When a component re-renders, React re-renders *all* its children, regardless of whether their props changed. This is expected behavior, not a bug. The question is whether that re-render is actually expensive.

**Fix:** Wrap the child in `React.memo` so it only re-renders when its props change:
```jsx
const Child = React.memo(function Child({ name }) { ... });
```

## 2. `React.memo` isn't helping because you pass new references every render
`memo` does a shallow comparison of props. If you pass inline objects, arrays, or functions, they're a *new reference* on every parent render, so the comparison always fails:
```jsx
// ❌ new object + new function every render → memo is useless
<Child style={{ color: 'red' }} onClick={() => doThing(id)} />
```

**Fix:** Stabilize the references:
```jsx
const handleClick = useCallback(() => doThing(id), [id]);
const style = useMemo(() => ({ color: 'red' }), []);
<Child style={style} onClick={handleClick} />
```

## 3. Context consumers re-render when the context value changes
If the child reads a context whose provider `value` is a fresh object each render, every consumer re-renders. `memo` won't save you here.

**Fix:** memoize the provider value with `useMemo`.

## How to find the real cause
Use the **React DevTools Profiler** — turn on "Highlight updates when components render," or record a profile. It tells you exactly *which* component re-rendered and *why* ("props changed: onClick", "hook changed", "parent rendered").
