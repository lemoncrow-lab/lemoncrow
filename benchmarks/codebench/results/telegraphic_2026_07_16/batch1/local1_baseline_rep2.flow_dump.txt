[assistant] Common causes, roughly in order of how often they're the culprit:

## 1. It's expected behavior (usually fine)
By default, **when a component re-renders, all of its children re-render** — regardless of whether their props changed. React doesn't bail out automatically. A re-render is not the same as a DOM update; React still diffs and often touches nothing. So the first question is: **is it actually a performance problem, or are you just seeing it in the Profiler?**

## 2. New reference identity on every render
This is the most common *fixable* cause. If the parent passes objects, arrays, or functions created inline, they're a **new reference every render**, which defeats memoization:

```jsx
// parent re-creates these every render
<Child style={{ color: 'red' }} onClick={() => doThing()} data={[1, 2, 3]} />
```

Fix with `useMemo` / `useCallback`:

```jsx
const handleClick = useCallback(() => doThing(), []);
const data = useMemo(() => [1, 2, 3], []);
```

## 3. The child isn't memoized
`useCallback`/`useMemo` only help if the child actually skips re-renders. Wrap it:

```jsx
const Child = React.memo(function Child(props) { ... });
```

`React.memo` does a shallow prop comparison and skips re-rendering when props are referentially equal — which is why #2 matters.

## The decision path

| Symptom | Action |
|---|---|
| Re-render is cheap / no real slowdown | Leave it — memoizing everything adds its own cost |
| Child is expensive & props genuinely unchanged | `React.memo` the child + stabilize props with `useMemo`/`useCallback` |
| `children` passed as prop | Lift them so the parent's state change doesn't recreate them |
| `Context` value changes | Memoize the provider `value`; split contexts |

## To diagnose which it is
- **React DevTools Profiler** → "Why did this render?" tells you exactly which prop/state/hook changed.
- Or the "Highlight updates when components render" setting to see the re-render cascade visually.

If you paste the parent + child component code, I can point to the specific line causing it — the answer is very different depending on whether it's inline props, context, or just an unmemoized child.
