[assistant] A debouncer on a search input delays running the search until the user *stops* typing for a short window (e.g. 300ms). Without it, you'd fire a search on **every keystroke**. Here's what that buys you:

**1. Fewer wasted requests**
If someone types "laptop," an un-debounced input fires 6 searches: "l", "la", "lap", "lapt", "lapto", "laptop." Only the last one matters. Debouncing collapses that into a single request once they pause.

**2. Less backend / API load**
Each keystroke-search hits your server, database, or a paid third-party API. Debouncing can cut request volume by ~80–90% on typical typing, which means lower cost and less strain on rate-limited services.

**3. Avoids out-of-order / stale results**
Rapid-fire requests can resolve out of order — the response for "lap" might arrive *after* "laptop" and overwrite the correct results with stale ones. Firing one request per pause sidesteps a whole class of race conditions (though you still want cancellation/latest-wins logic for robustness).

**4. Smoother UX**
The results panel doesn't flicker and re-render on every character. The user sees results settle once they've expressed a complete-enough query.

**5. Less client-side work**
For local/in-memory filtering over a large dataset, re-filtering and re-rendering on every keystroke can jank the UI. Debouncing throttles that work.

---

**Quick nuance — debounce vs. throttle:**
- **Debounce** = wait until activity *stops*, then fire once. Ideal for search-as-you-type, because you care about the *final* query, not intermediate states.
- **Throttle** = fire at most once per N ms *during* activity. Better for things like scroll or resize where you want periodic updates.

**Common gotcha:** pick the delay carefully. Too short (~100ms) and you barely save anything; too long (~800ms+) and the input feels laggy. **250–400ms** is the usual sweet spot. And debouncing doesn't replace request cancellation — for network searches you still want to cancel or ignore superseded in-flight requests to guarantee the latest query wins.
