```tsx
import { Component, type ErrorInfo, type ReactNode } from "react";

export class ErrorBoundary extends Component<Props, State> {
  state: State = { error: null };

  static getDerivedStateFromError(error: Error): State {
    return { error };               // → switch to fallback on next render
  }

  componentDidCatch(error: Error, info: ErrorInfo): void {
    console.error("ErrorBoundary caught:", error, info.componentStack);
    this.props.onError?.(error, info);   // hand off to Sentry/Datadog/etc.
  }

  reset = () => this.setState({ error: null });  // retry → re-render children
  // ...render(): fallback with a "Try again" button, else children
}
```

```tsx
  fallback={(err, reset) => (

      <button onClick={reset}>Retry</button>

```

**Key points**

- **Must be a class.** There is no hook for `getDerivedStateFromError`/`componentDidCatch`. The common workaround is to use the `react-error-boundary` library, which wraps a class internally.
- **Three responsibilities, mapped to lifecycle:** `getDerivedStateFromError` flips state to render the fallback; `componentDidCatch` logs the error + `componentStack`; the retry button calls `reset()` to clear the error so children re-render.
- **What it does *not* catch:** errors in event handlers, `setTimeout`/promises/async code, SSR, and errors thrown in the boundary itself. Wrap those in a manual `try/catch`.
- **Retry caveat:** if the underlying condition still fails, the child will just re-throw. For a hard reset (e.g. refetch data), give the wrapped subtree a changing `key` or pass a `resetKeys`-style prop.
