[assistant] Created `src/ErrorBoundary.tsx`. It's a standard class-based React error boundary (the only way to catch render errors in React).

**What it does:**

- **Catches render errors** — `getDerivedStateFromError` flips state to show the fallback; `componentDidCatch` receives the error + component stack.
- **Fallback UI with retry** — a default `role="alert"` panel showing the error message and a "Try again" button. `retry()` clears the error state so React re-renders the children (a fresh attempt).
- **Logs details** — `console.error` with the error and `componentStack`, plus an optional `onError` hook to forward to a service like Sentry.

**Usage:**

```tsx
<ErrorBoundary>
  <MyApp />
</ErrorBoundary>

// or with a custom fallback + reporter:
<ErrorBoundary
  onError={(err, info) => reportToSentry(err, info)}
  fallback={(error, retry) => (
    <div>
      <p>Oops: {error.message}</p>
      <button onClick={retry}>Retry</button>
    </div>
  )}
>
  <MyApp />
</ErrorBoundary>
```

**One caveat to be aware of:** like all React error boundaries, it only catches errors thrown during rendering/lifecycle — not those from event handlers, `setTimeout`, or async code. I documented that in the component's doc comment.
