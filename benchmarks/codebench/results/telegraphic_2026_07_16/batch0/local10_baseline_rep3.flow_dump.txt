- **Catches render errors** — `getDerivedStateFromError` flips into an error state so the fallback renders instead of the crashed subtree.

**Usage:**

A couple of things worth knowing given the requirements:

- **Scope of "render errors":** error boundaries catch errors during render, lifecycle methods, and child constructors — but *not* event handlers, async code (`setTimeout`/promises), or the boundary itself. I noted this in the file's docblock since it's the most common surprise. Those need regular `try/catch`.
- **Retry caveat:** clicking retry re-renders the same children. If the underlying cause (bad prop, failing data) is still present, it'll error again immediately. The `onReset` prop lets the parent reset that state (refetch, change a key) alongside the retry.
