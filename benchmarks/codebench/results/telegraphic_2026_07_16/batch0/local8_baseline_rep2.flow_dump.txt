[assistant] Done. Here's how the size minimization works:

**Multi-stage layout (4 stages):**
- **`deps`** — `npm ci` with all deps (needs `typescript` etc. to compile).
- **`build`** — compiles TS → JS via `npm run build`.
- **`prod-deps`** — a *separate* `npm ci --omit=dev` so dev dependencies (TypeScript, `@types/*`, linters) never reach the final image.
- **`runtime`** — copies only prod `node_modules` + compiled `dist/`. None of the build tooling, source `.ts` files, or dev deps are carried forward.

**Key size/quality choices:**
- `node:22-alpine` base — much smaller than the default Debian-based image.
- Manifests copied before source, so the `npm ci` layers stay cached until dependencies actually change.
- Splitting `prod-deps` from `deps` is the main win — the final image gets a pruned dependency tree instead of the full dev toolchain.
- `.dockerignore` keeps the build context lean and prevents host `node_modules` from clobbering the installed ones.
- Runs as the non-root `node` user with `NODE_ENV=production`.

**Assumptions you may need to tweak** (repo was empty):
- Build script is `npm run build` and emits to `dist/` with entrypoint `dist/index.js` — adjust `outDir`/paths to match your `tsconfig.json`.
- App listens on `3000` — change `EXPOSE` to your port.
- Pinned Node 22 LTS; a `package-lock.json` must exist for `npm ci`.

If you want an even smaller image, the next step would be a `node:22-alpine` → `gcr.io/distroless/nodejs22` runtime, or bundling to a single file with esbuild/`ncc` to drop `node_modules` entirely.
