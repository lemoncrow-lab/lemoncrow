[assistant] Done. Key size-minimizing decisions:

**Three stages, so build tooling never reaches the final image:**
- **`build`** — installs *all* dependencies (`npm ci`) including `typescript` and other devDependencies, then runs `npm run build` to compile TS → JS.
- **`deps`** — a separate clean install of **production-only** deps (`npm ci --omit=dev`). Kept separate from `build` so the pruned `node_modules` isn't contaminated by devDependencies.
- **`runtime`** — copies in only the compiled `dist/` and the production `node_modules`. No source `.ts`, no compilers, no dev tooling.

**Other choices worth noting:**
- `node:22-alpine` base — Alpine keeps the runtime layer small. Swap the tag to your Node version. (If a native module fails to build against musl, use `node:22-slim` instead.)
- Manifests copied before source so the `npm ci` layers stay cached until `package-lock.json` actually changes.
- Runs as the non-root `node` user; `ENV NODE_ENV=production`.
- `CMD ["node", "dist/index.js"]` rather than `npm start` — one fewer process, and signals (SIGTERM) reach Node directly for clean shutdown.

**You'll likely want to adjust:**
1. **`npm run build`** must exist and output to `dist/` (typically `"build": "tsc"` with `"outDir": "dist"` in `tsconfig.json`).
2. **Entry point** — change `dist/index.js` if yours differs.
3. Add a **`.dockerignore`** to keep `node_modules`, `.git`, and `dist` out of the build context:

