[assistant] Done. Created `ErrorBoundary.tsx` — a class-based error boundary (the only way to catch render errors in React, since there's no hook equivalent for `componentDidCatch`).

**What it does:**
- **Catches render errors** via `getDerivedStateFromError`, which flips state to show the fallback.
- **Fallback UI with retry** — an accessible `role="alert"` panel showing the error message plus a "Try again" button. `reset()` clears the error state so the subtree re-mounts and re-attempts the render.
- **Logs the error** in `componentDidCatch` (`console.error` with the error + component stack), and forwards to an optional `onError` prop so you can wire in Sentry/Datadog.

**Usage:**

```tsx
<ErrorBoundary onError={(err, info) => reportToSentry(err, info)}>
  <App />
</ErrorBoundary>
```

Optionally supply a custom fallback:

```tsx
<ErrorBoundary fallback={(error, reset) => <MyFallback error={error} onRetry={reset} />}>
  <App />
</ErrorBoundary>
```

Two things worth knowing about React error boundaries in general:
- They **don't** catch errors in event handlers, async code, SSR, or the boundary itself — only errors thrown during render/lifecycle of the tree below.
- A bare retry only helps if the underlying cause is transient (e.g. a flaky fetch). For a hard bug, the render will just fail again — often you'll pair retry with a `resetKeys`-style prop or a navigation reset.
