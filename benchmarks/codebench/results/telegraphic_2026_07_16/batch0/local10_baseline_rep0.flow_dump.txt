**What it does:**
- **Catches render errors** — implements `getDerivedStateFromError` (switch to fallback) and `componentDidCatch` (side effects). These are class-only lifecycles; there's no Hook equivalent, so this must be a class component.

**Extras:**
- `fallback` render-prop for custom UI: `(error, retry) => ReactNode`.
- `resetKeys` — auto-resets the boundary when a watched value (e.g. route) changes, so a stale error doesn't persist.

**Usage:**
```tsx
<ErrorBoundary onError={(e, info) => logToService(e, info)} resetKeys={[location.pathname]}>

</ErrorBoundary>
```

Two caveats worth knowing: error boundaries only catch errors during **render/lifecycle/constructors** of descendants — not event handlers, async code, or the boundary itself. And plain retry only recovers if the underlying cause is transient; if the same input re-throws, pair `retry` with `resetKeys` or a state change that fixes the input.
