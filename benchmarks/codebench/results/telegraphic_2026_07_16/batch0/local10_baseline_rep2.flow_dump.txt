[assistant] Created `ErrorBoundary.tsx`. It covers all three requirements:

- **Catches render errors** — via `getDerivedStateFromError` (sets fallback state) and `componentDidCatch` (side effects). It's a class component because React has no hook equivalent for these lifecycle methods.
- **Fallback UI with retry** — a default styled `role="alert"` panel showing the error message and a **Try again** button. The `retry` callback clears the error state so React re-renders the children.
- **Logs error details** — `componentDidCatch` logs the error plus `componentStack` to the console and forwards them to an optional `onError` prop for wiring up real telemetry.

Usage:

</ErrorBoundary>

Two caveats worth knowing: error boundaries **don't** catch errors in event handlers, async code, or SSR — only render/lifecycle/constructor errors in the tree below them. And a plain retry re-renders the same children, so if the underlying cause persists it'll error again; often you'll pair `retry` with a state reset or a `resetKeys`-style prop upstream.
