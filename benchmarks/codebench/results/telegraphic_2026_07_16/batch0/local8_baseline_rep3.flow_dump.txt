[assistant] Created a three-stage `Dockerfile`. How it minimizes the final image:

**Stage 1 (`build`)** — installs all dependencies (including `typescript` and `@types/*`) with `npm ci`, then runs `npm run build` to compile TS → JS.

**Stage 2 (`deps`)** — a clean install of production deps only (`npm ci --omit=dev`), so no compiler or type packages leak into runtime.

**Stage 3 (`runtime`)** — starts from a fresh `node:22-alpine` and copies in *only* the pruned `node_modules`, the compiled `dist/`, and `package.json`. None of the source `.ts`, dev tooling, or build cache carries over.

Key size/hardening choices:
- **Alpine base** — ~10× smaller than the default Debian-based `node` image.
- **Manifest-first copy + `npm ci`** — deterministic installs and cached dependency layers across rebuilds.
- **`--omit=dev` + `npm cache clean`** — drops devDependencies and the npm cache.
- **Non-root `node` user** and **`dumb-init`** as PID 1 for correct signal handling.

- The build assumes `tsconfig.json` emits to `dist/` and your entry is `dist/index.js` — change the `CMD` / `COPY --from=build /app/dist` if your `outDir` or entrypoint differs.
