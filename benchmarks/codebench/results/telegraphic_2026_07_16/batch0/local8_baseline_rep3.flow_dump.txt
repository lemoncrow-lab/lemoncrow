[assistant] I created a `Dockerfile` with four stages that keep the final image lean.

**How it minimizes size:**

- **Separate build vs. runtime images** — `tsc` and all devDependencies live only in the `build` stage and never reach the final image.
- **Clean production `node_modules`** — the `prod-deps` stage runs `npm ci --omit=dev` in isolation, so the runtime copies a `node_modules` that was *never* polluted with dev packages (pruning in place leaves cruft behind; a fresh install doesn't).
- **Final stage copies only three things** — production deps, compiled `dist/`, and `package.json`. No source `.ts`, no tsconfig, no build tooling.
- **`node:22-alpine` base** — Alpine is ~10x smaller than the default Debian image.

**Things to adjust for your actual project:**

- **Base image tag** — pin to the Node version you target. If you use native modules that don't build on musl/Alpine, switch to `node:22-slim` (Debian).
- **`npm run build`** — assumes a `build` script running `tsc`. Confirm it exists in `package.json`.
- **Output path** — assumes TypeScript emits to `./dist`. Match your `tsconfig.json` `outDir`.
- **Entrypoint** — `dist/index.js` is a guess; set it to your compiled entry file.
- **`EXPOSE 3000`** — change to your app's port (or drop it if not a server).
