[assistant] Done. `Dockerfile` uses four stages to keep the final image small.

**How it minimizes size:**

- **Compiler and devDependencies never reach the final image.** TypeScript, type definitions, and build tooling live only in the `build` stage. The `runner` copies just compiled `dist/` output plus production `node_modules`.
- **Separate `prod-deps` stage** runs `npm ci --omit=dev`, so the runtime `node_modules` contains only what the app needs to run — not the full dev install used to compile.
- **Alpine base** (`node:22-alpine`) is a small runtime.
- **Layer caching:** manifests are copied before source, so `npm ci` only re-runs when `package.json`/`package-lock.json` change.

**Assumptions you may need to adjust:**

| Item | Assumed value | Change if… |
|------|---------------|-----------|
| Build script | `npm run build` | Your `package.json` uses a different script |
| Output dir | `dist/` | Your `tsconfig.json` `outDir` differs |
| Entry point | `dist/index.js` | Your main file differs |
| Port | `3000` | Your app listens elsewhere (or it's not a server) |
| Node version | `22` | You need a different runtime |
