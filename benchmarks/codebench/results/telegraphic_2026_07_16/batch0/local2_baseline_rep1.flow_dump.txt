## The bug: seconds vs. milliseconds

JWT's `exp` claim is expressed in **seconds** since the Unix epoch (per RFC 7519 — it's a "NumericDate"). But `Date.now()` returns **milliseconds**.

So a comparison like this:

```js
if (Date.now() >= decoded.exp) {   // BUG
  throw new Error('token expired');
}
```

compares, e.g., `1752624000000` (ms, now) against `1752627600` (seconds, exp). `Date.now()` is ~1000× larger, so `Date.now() >= decoded.exp` looks "already expired"... except people usually write it the other way (`decoded.exp > Date.now()` → "still valid"), and *that* is always true until the year ~53,000. Either way the magnitude mismatch makes the check meaningless, and expired tokens sail through.

## The fix

Convert to the same unit. Compare in **seconds**:

```js
const nowSeconds = Math.floor(Date.now() / 1000);
if (decoded.exp <= nowSeconds) {
  return res.status(401).json({ error: 'token expired' });
}
```

## Better fix: don't hand-roll the check

`jsonwebtoken.verify()` already validates `exp` for you (in the correct units) and throws a `TokenExpiredError`. Manual expiry checking is a common source of exactly this bug. Prefer:

```js
const jwt = require('jsonwebtoken');

function auth(req, res, next) {
  const token = req.headers.authorization?.split(' ')[1];
  if (!token) return res.status(401).json({ error: 'no token' });

  try {
    req.user = jwt.verify(token, process.env.JWT_SECRET);
    // optionally: { algorithms: ['HS256'], clockTolerance: 5 }
    next();
  } catch (err) {
    if (err.name === 'TokenExpiredError') {
      return res.status(401).json({ error: 'token expired' });
    }
    return res.status(401).json({ error: 'invalid token' });
  }
}
```

Notes:
- `verify()` throws on expiry, bad signature, and malformed tokens — so pin `algorithms` to prevent `alg: none` / algorithm-confusion attacks.
- `clockTolerance` (seconds) covers small server clock skew if you were seeing borderline failures.
- If you're using `jwt.decode()` instead of `jwt.verify()` anywhere in this path, that's a second serious bug — `decode()` does **no** signature or expiry validation at all.

**Root cause in one line:** `exp` is in seconds, `Date.now()` is in milliseconds — either divide `Date.now()` by 1000, or let `jwt.verify()` handle it.
