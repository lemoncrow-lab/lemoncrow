## What's wrong

A **units mismatch**. Per RFC 7519, a JWT's `exp` claim is a Unix timestamp in **seconds**. `Date.now()` returns **milliseconds** — about 1000× larger.

So a real `exp` (~`1.75e9`) is always tiny compared to `Date.now()` (~`1.75e12`). Written the usual way, that lets every token pass:

```js
// "is the token still valid?" — exp is always < Date.now(), so this is never true
if (decoded.exp > Date.now()) { /* valid */ }
else return res.status(401)...   // this branch also mis-fires depending on direction
```

Depending on which direction your comparison runs, the seconds-vs-ms skew either rejects everything or (your symptom) never flags anything as expired.

The other classic cause of *this exact symptom* — expired tokens passing rather than everything being rejected:

- **`exp` is `undefined`**, so the comparison becomes `something > NaN`, which is always `false` → nothing is ever treated as expired. This happens if you used `jwt.decode()` (which does **not** verify) instead of `jwt.verify()`, or the token was signed without an expiry.

## The direct fix

Normalize to the same unit and reject when now ≥ expiry:

```js
const nowSeconds = Math.floor(Date.now() / 1000);
if (!decoded.exp || decoded.exp <= nowSeconds) {
  return res.status(401).json({ error: 'Token expired' });
}
// equivalently: if (!decoded.exp || decoded.exp * 1000 <= Date.now()) { ... }
```

## Better: don't hand-roll it

`jsonwebtoken`'s `verify` already checks `exp` (with correct units), the signature, and `nbf`, and throws a typed error — so the units bug becomes impossible and you also get signature validation (which a manual `decode` + `exp` check skips entirely):

```js
const jwt = require('jsonwebtoken');

function requireAuth(req, res, next) {
  const token = (req.headers.authorization || '').replace(/^Bearer\s+/i, '');
  try {
    req.user = jwt.verify(token, process.env.JWT_SECRET);
    next();
  } catch (err) {
    if (err.name === 'TokenExpiredError') return res.status(401).json({ error: 'Token expired' });
    return res.status(401).json({ error: 'Invalid token' });
  }
}
```
