## How it minimizes size

The image is built in **three stages**, and only the last one is shipped:

1. **`deps`** — runs `npm ci` to install *all* dependencies (including dev deps like `typescript`), cached on the manifests so it only re-runs when `package.json`/`package-lock.json` change.
2. **`build`** — compiles TypeScript with `npm run build`, then `npm prune --omit=dev` drops the dev dependencies.
3. **`runtime`** — starts from a fresh `node:22-alpine` and copies in **only** the pruned `node_modules`, the compiled `dist/`, and `package.json`. The TypeScript compiler, source `.ts` files, tests, and build tooling never reach the final image.

## Other size/quality choices

- **`alpine` base** — ~50 MB vs ~1 GB for the full Debian `node` image.
- **`.dockerignore`** — keeps `node_modules`, `.git`, tests, and docs out of the build context (faster builds, avoids leaking a host `node_modules`).
- **BuildKit cache mount** on `/root/.npm` — speeds rebuilds without adding a layer.
- **Non-root `node` user** and `NODE_ENV=production` for a safer, leaner runtime.

## You'll likely need to adjust

- **`node dist/index.js`** — change to your actual entrypoint.
- **`npm run build`** — assumes a `build` script running `tsc`; and I assumed `outDir: "dist"` in your `tsconfig.json`. Align the `COPY --from=build /app/dist` path if yours differs.
- **`EXPOSE 3000`** — set to your app's port (or remove for a non-server app).
- **`node:22-alpine`** — pin to whatever Node version your app targets; consider a digest pin (`node:22-alpine@sha256:...`) for fully reproducible builds.
