What it does:

- **Catches render errors** — `getDerivedStateFromError` flips state to show the fallback; `componentDidCatch` captures the component stack. (Error boundaries must be class components — there's no hook equivalent for these lifecycles.)
- **Fallback UI + retry** — default alert-styled panel with the error message and a **Retry** button that clears error state so children re-render. A `fallback(error, retry, errorInfo)` render prop overrides the default.
- **Logs details** — calls your `onError(error, errorInfo)` if provided, otherwise `console.error`.

Usage:

```jsx
<ErrorBoundary onError={(err, info) => reportToSentry(err, info)}>
  <App />
</ErrorBoundary>
```

Two things worth knowing:
- Error boundaries only catch errors during render/lifecycle of **children** — not event handlers, async code, or errors thrown by the boundary itself.
- Plain retry re-renders the same tree; if the underlying cause persists it'll error again. For data-fetch failures, pair the retry with a `key` prop reset or refetch.
