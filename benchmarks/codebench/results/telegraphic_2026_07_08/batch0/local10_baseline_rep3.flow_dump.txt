[assistant] Created `ErrorBoundary.jsx`. Key points:

**Why a class component** — Error boundaries require `getDerivedStateFromError` / `componentDidCatch`, which have no Hook equivalent. There's no way to build one purely with function components.

**How it works:**
- `getDerivedStateFromError` flips state to render the fallback instead of the crashed subtree.
- `componentDidCatch` handles the side effects: logs to console and forwards to an optional `onError` prop for a real service (Sentry, etc.).
- The **retry button** calls `reset()`, which clears the error state → React re-renders `children`, retrying the failed subtree.

**Usage:**

```jsx
<ErrorBoundary onError={(err, info) => reportToSentry(err, info)}>
  <MyApp />
</ErrorBoundary>

// Or with a custom fallback:
<ErrorBoundary fallback={({ error, reset }) => (
  <MyFallback error={error} onRetry={reset} />
)}>
  <MyApp />
</ErrorBoundary>
```

**Two caveats worth knowing:**

1. **Boundaries only catch render-phase errors** — not event handlers, `setTimeout`/promises, SSR, or errors thrown by the boundary itself. For those, use a regular `try/catch`.
2. **Plain retry may loop.** If the error is deterministic (e.g., bad data in a parent), clicking "Try again" re-renders the same broken tree and crashes again. For those cases, pair retry with a `resetKeys` prop (reset when a dependency changes) or fix the upstream cause. If you'd rather not maintain this yourself, [`react-error-boundary`](https://github.com/bvaughn/react-error-boundary) is the well-established library covering these patterns.
