## plugin:lemoncrow:lemoncrow
LemonCrow replaces the grep→read→re-read loop.
- Lead with `code_search`: one call = ranked matches (top source inline, the rest as precise path:Lx-Ly pointers) + related_symbols + candidate_files. Inline source = already read. Never shell-grep workspace code or re-verify indexed results.
- Known path/symbols →  batch `read`: ONE call, files=[...], exact :Lx-Ly ranges, never the same file twice. Never cat/sed/head/tail.
- ALL edits in ONE `edit` edits[] array; prefer {path: 'f.py:Lx-Ly', new} over old/new.
- `bash` = execution only (tests, git, builds).
- Large output → a file, never inline prose.

[assistant] ```dockerfile
# syntax=docker/dockerfile:1

# ---- Stage 1: deps (all deps incl. dev, for building) ----
FROM node:22-alpine AS deps
WORKDIR /app
COPY package.json package-lock.json ./
RUN npm ci

# ---- Stage 2: build (compile TS -> JS) ----
FROM node:22-alpine AS build
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY . .
RUN npm run build          # tsc -> ./dist

# ---- Stage 3: prod deps only ----
FROM node:22-alpine AS prod-deps
WORKDIR /app
COPY package.json package-lock.json ./
RUN npm ci --omit=dev

# ---- Stage 4: runtime (minimal) ----
FROM node:22-alpine AS runtime
ENV NODE_ENV=production
WORKDIR /app
COPY --from=prod-deps /app/node_modules ./node_modules
COPY --from=build /app/dist ./dist
COPY package.json ./
USER node
EXPOSE 3000
CMD ["node", "dist/index.js"]
```

Size levers, in effect:
- `alpine` base → ~50MB vs ~1GB `node:22`.
- prod `node_modules` built in isolated stage with `--omit=dev` → no tsc/types in final.
- final image = base + prod deps + `dist` only; source, dev deps, build cache excluded.

Assumptions:
- `npm run build` → `tsc` emitting `./dist`, entry `dist/index.js`.
- listens on 3000.

Also add `.dockerignore`:
```
node_modules
dist
.git
npm-debug.log
```

Adjust: pin Node major to yours; swap entry/port to match `package.json`. Smaller still → `node:22-alpine` + `distroless`/`gcr.io/distroless/nodejs22` for runtime, or add `RUN npm prune --production` variants — but the 4-stage split above is the standard minimal.
