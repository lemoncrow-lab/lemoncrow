[assistant] Pool = reuse open connections. Use `pg` lib (node-postgres).

```js
const { Pool } = require('pg');

const pool = new Pool({
  host: process.env.PGHOST,
  port: 5432,
  user: process.env.PGUSER,
  password: <redacted>,
  database: process.env.PGDATABASE,

  max: 20,                       // max clients in pool
  min: 0,                        // min idle clients
  idleTimeoutMillis: 30000,      // close idle client after 30s
  connectionTimeoutMillis: 5000, // fail if no connection in 5s
  maxUses: 7500,                 // retire client after N queries (dodge mem leak / stale conn)

  // per-query server-side timeouts
  statement_timeout: 10000,          // kill query after 10s
  query_timeout: 10000,              // client-side query timeout
  idle_in_transaction_session_timeout: 10000, // kill idle-in-txn
});
```

Key knobs:

- **`connectionTimeoutMillis`** — cap wait for free/new connection. No set = hang forever when pool exhausted.
- **`idleTimeoutMillis`** — reap idle clients, free DB resources.
- **`statement_timeout`** — server kills slow query. Protect against runaway.
- **`max`** — total connections. Keep under Postgres `max_connections` divided across app instances.

Error handling — pool emit `error` on idle client that dies (network drop, DB restart). Must listen or process crash:

```js
pool.on('error', (err) => {
  console.error('idle client error', err);
  // do NOT throw. pool auto-removes bad client.
});
```

Query with proper release:

```js
async function getUser(id) {
  const client = await pool.connect();
  try {
    const { rows } = await client.query('SELECT * FROM users WHERE id = $1', [id]);
    return rows[0];
  } finally {
    client.release(); // ALWAYS release, even on throw. Leak = pool exhaust
  }
}
```

For simple one-shot query, skip manual connect — `pool.query()` auto-acquire+release:

```js
const { rows } = await pool.query('SELECT * FROM users WHERE id = $1', [id]);
```

Graceful shutdown — drain pool on exit:

```js
process.on('SIGTERM', async () => {
  await pool.end(); // wait active queries, close all
  process.exit(0);
});
```

Gotchas:
- `client.release(true)` = destroy client (pass on suspected-bad connection).
- Retry transient connect fail (DB not ready at boot) with backoff.
- `max` too high = overload Postgres. Count all app replicas.
