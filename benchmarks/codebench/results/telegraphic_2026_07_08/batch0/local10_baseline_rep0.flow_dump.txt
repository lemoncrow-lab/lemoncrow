[assistant] Created `ErrorBoundary.tsx`. Since error boundaries require a class component (React has no hook equivalent for `getDerivedStateFromError`/`componentDidCatch`), this is a class-based component covering all three requirements:

- **Catches render errors** — `getDerivedStateFromError` captures the thrown error and switches to the fallback on the next render.
- **Fallback UI with retry** — a default `role="alert"` panel showing the error message plus a "Try again" button. `retry` clears the error state so `children` re-mount and re-render.
- **Logs error details** — `componentDidCatch` logs the error and component stack via `console.error`, and forwards them to an optional `onError` prop for external reporting (Sentry, etc.).

It's also customizable via a `fallback` render prop.

Usage:

```tsx
import { ErrorBoundary } from "./ErrorBoundary";

<ErrorBoundary onError={(err, info) => reportToSentry(err, info)}>
  <YourApp />
</ErrorBoundary>

// or with a custom fallback:
<ErrorBoundary fallback={(err, retry) => <MyFallback error={err} onRetry={retry} />}>
  <YourApp />
</ErrorBoundary>
```
