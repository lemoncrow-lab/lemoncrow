[assistant] Created `ErrorBoundary.tsx` — a class-based React error boundary since `getDerivedStateFromError`/`componentDidCatch` (render-error catching) only work in class components, not hooks.

Key points:
- **`getDerivedStateFromError`** updates state to render the fallback on the next render.
- **`componentDidCatch`** logs the error and component stack via `console.error`, and forwards both to an optional `onError` prop for integration with logging services (Sentry, etc.).
- **Retry** just clears the stored error, causing children to remount and re-render — if the same broken state caused the crash it'll fail again, which is expected (retry is for transient errors, not primarily to fix code bugs).
- **`fallback`** prop lets callers render custom UI (receiving the error and retry function) instead of the default box.

Since this repo is an intentionally empty benchmark scratch repo (per its README) with no existing app to wire it into, I dropped the component at the repo root as a self-contained file.
